Asymptotic Notation

Asymptotic Analysis:



> How the RUNNING_TIME of an algorithm increases with the increase in INPUT_SIZE.
> To find the rate_of_growth_runnig_time of an algorithm, BIG O Notation is used.

> It is the most commonly used mathematical tool to measure efficiency of an algorithm.
> Notation, f(n) → It is a function which we have but don't know which class of function it belongs to so that we can know its rate of growth. But we are sure that it will lie to one of the classes of g(n).
g(n) →It is a set of known functions (like (logn, n, etc) whose rate of growth we already know in order) where each function represents a class for Asympototic Notations like Big Oh, Big Omega, Theta .


1) BIG O NOTATION:
> Mathematically,
f(n) is O(g(n) if there exists constant c and input n0 such that
f(n) <= c*g(n) for all n > n0 which means, g(n) always grows faster than f(n) so it becomes upperBound for f(n).
> NOTE: There can be multiple functions in the g(n) class which will grow faster than f(n) but we will try to stick with TIGHT(closer/least) UPPER BOUND from g(n) classes because it will be more informative .

* TIGHT UPPER BOUND & LOOSE UPPER BOUND

Since, 5n2 + 4n + 8 <= 6n2 or 7n2 , 6n3, 7n3 and so on which are greater in the class which means f(n) = c*n2 which implies c = 6,7,8...so on and n2 = g(n). We know f(n)=O(g(n)), thus we have f(n) = O(n2) which says that UPPER_GROWTH RATE OF f(n) IS O(n2).

Some set of standard functions which rate of growth we already know which makes up g(n) class.




2) Big Omega (_^_) Notation : f(n) = _^_(g(n) if f(n) >= g(n) which gives lower bound and indicats LOWER_GROWTH_RATE which means function will decrease with rate
3) Theta(0) Notation: f(n) = 0(g(n)) if g(n)<= 0(f(n)) <=g(n) which gives average bound and indicats AVARAGE_GROWTH_RATE which means the function can either increase or decrease with rate.

>> Shorthand, We can only find Theta Notation for a function f(n) if lower bound / omega Notation of f(n) = upper bound/Big O notation.

If Theta can be found, it is better to go for it
because theta noation is more precise(nearest to the actual growth rate), if theta cannot be found then go for other notations.
for e.g we cannot find Theta notaion for factorial function because it has O(n2) and _^_(1) which are not equal.

NOTE: Wherever we are unable to get average rate or THETA notation of a function then we go for either Big Oh or Big Omega Notation.





So Fastest Growing term(like 3n2) is taken into consideration and other(slow growing term, constanats, coefficients) are discarded.
Note: Base of the logarithmic is not important for finding order of the function.
