*** 3 Very important PROPERTIES used in CSS Layout Design: ***



1. DISPLAY: how one html element should interact with its neighbour element while laying out on the webpage.


i.inline

iii.block
iv.table
v.float

viii.none

i) inline-elements: By default these elements have display:inline property.


<span>
<a>
<img>
>>>>>> displays elements in line and doesn't respect width and height property.(for text)
But it does respect padding and height but also to left and right only

ii) display:inline-block elements


>>>>>> displays elements in line and also respect width and height property.

iii) block-elements: By default these elements have display:block property



<div>
<h1> - <h6>
<p>
<form>
<header>
<footer>
<section>
>>>>>> respect width and height property.(for section/division in webpage)

iv. table


>>>>>> for keeping data in two dimensional form.
table,th,tr,td (heading,row,data) are used to populate a 2D table.

v. float


<p><img>text.....</p>
Since p is block element and img is inline element, img will stay in line with p which will waste space equal to image height.
So float comes into picture.
>>>>>> Especially used to wrap text around image by floating image either towards left or right the element.(inline-element)
float:left; i sit toward left and other can sit around me.
clear: left; i will not sit around any the left-floated elements. Basically doesn't allow to float around floated item.
Clearfix: if a floated element is overflowed, then it is used to fix.
.clearfix::after {
content: "";
clear: both;
display: table;
}

vi.FlexBox


>>>>>> Modern CSS Module for one dimensional layout Design.
One group of elements can be controlled in any one direction using flexbox.

vii.Grid:


>>>>>> Modern CSS Module for TWO dimensional layout Design.
One group of elements can be controlled in two direction at a time using grid


viii) display:none: display:none on any element will remove it from the webpage and space will be freed


visibility:hidden: will hide the element on the webpage but space will still be occupied.

2. Postiton property:



1. static:


>>> According to the normal flow /order of elements in source code. By the way it is default position.
* It doesn't respect top, right, bottom, left properties.

2. relative:


>>> Causes normally(default) positioned element to adjust away from its normal position/flow/order.
* It respects top, right, bottom, left properties.

3. fixed:


>>> position an element with respect to the viewport. So the element always stays at same position even if the page is scrolled.
* It also respects top, right, bottom, left properties.

4. absolute:


>>> position an element with respect to its immediate positioned(either relative or absolute) parent_div.Also
if the parent_div is not positioned, the element use body element position and moves along with page scrolling
* It also respects top, right, bottom, left properties.

5. sticky:


>>> sticks an element with respect to the scroll position.It differs from fixed as its position is not always fixed. Consider a
scenerio when an element is postioned at the bottom and sticks when reaches to the top.
It the page is being scrolled vertically, then either top or bottom property must be specified.
It the page is being scrolled horizontally, the either left or right property must be specifited.
NOTE: any of its parent div must not overflow:hidden property, otherwise it will not work
* e.g postion:sticky;
top:10px;
which means While scrolling if the element is 10px away from top, make it sticky.

3. Stack-order:(z-axis of the box)


>>> The z-index property specifies the stack order of an element.
An element with greater stack order is always in front of an element with a lower stack order.
* z-index only works on positioned elements (position: absolute, position: relative, position: fixed, or position: sticky).
* It won't work if element is nomrmally positioned(default/static).
* Any child element within the current element will use the parentâ€™s Z-index as a starting point.for more refer stacking context
